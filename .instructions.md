Agent Instructions: Prefer workspace MCP for documentation searches

Repository rule: persistently use `mcp_monorepo-mark_search_documentation` for any prompt that asks to search documentation, workspace guidance, or model/integration reference material.

When a user asks to search documentation, reference material, or implementation guidance about workspace tooling, Codex skills, Copilot, Cline, MCP, or models in general, follow this order of operations:

1. Call the MCP search tool `mcp_monorepo-mark_search_documentation` with the user's query as the `query` parameter and the repository root as the workspace. Use the `subproject` parameter only when the user specifies a subfolder.

2. If the MCP returns relevant results, summarize them concisely and cite the original file paths and line ranges returned by the MCP. Always prioritize MCP results over ad-hoc greps or manual file reads.

3. If the MCP returns no results or the server is unavailable, fall back to reading matching files in the workspace using the file reader tools (`read_file`, `file_search`, `grep_search`). Before performing broad workspace reads, inform the user and ask for confirmation if the search will scan many files.

4. Never query external web search engines for internal repository docs unless the user explicitly asks for web research.

5. For queries about `codex`, `copilot`, `cline`, `mcp`, or `model(s)`, always attempt the MCP first. This includes requests like "search docs for codex skills", "where are copilot adapters defined", "how does cline register MCP servers", or "show model integration docs".

6. When returning MCP-based results, provide a short summary (2–5 bullets) and include file links with paths and line ranges so the user can inspect the source directly.

7. If MCP is unavailable and the user requests me to proceed without it, explicitly confirm the fallback choice and proceed only after user consent.

Rationale: The MCP has an index tuned to repository docs and policy guidance; using it first yields higher-quality, context-aware answers and preserves consistency across agents.

Example prompts that should trigger MCP use:
- "Search the Codex skills for deployment notes"
- "Find where Copilot adapters are documented"
- "Show me docs about how MCP servers are registered"
- "Search model integration guides in the repo"

Placement and usage notes:
- Keep this file at the repository root as `.instructions.md` so local agents and automation can use it as a first-class instruction source.
- If you want me to also update `.codex/` skills or add enforcement in other locations, say which files to update and I'll apply the changes.
